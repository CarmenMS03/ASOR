***EJ2****
ping6 fe80::a00:27ff:fe88:453f%enp0s3 (dirección de enlace local del enp0s3 de vm_2) saliendo por enp0s3 de vm_1

--> añadir el nombre de la interfaz DE ORIGEN al usar direcciones de enlace local. Se indica la interfaz de origen y el protocolo encontrara la interfaz de destino asociada a esa dirección (que aunque el host al que vayamos tenga varias interfaces con la misma dirección SOLO UNA pertenece la mismo enlace--> ámbito local)
Esto fija el enlace en el que estamos: Aunque un router tenga varias interfaces con la misma dirección de enlace local, solo una de ellas estará conectada al mismo enlace que el host emisor, por lo que el paquete llegará únicamente a esa interfaz.

Las direcciones IPv6 de enlace local deben ser únicas dentro de un enlace. Interfaces de distintos nodos no pueden compartir la misma dirección link-local en el mismo enlace. Sin embargo, un mismo nodo puede asignar la misma dirección link-local a varias interfaces, ya que cada una pertenece a un enlace distinto.
En IPv6, la dirección link-local identifica una interfaz dentro de un enlace, no un host globalmente.
